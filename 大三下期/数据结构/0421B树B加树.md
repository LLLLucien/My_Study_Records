# B树与B+树

## 二叉搜索树（BST）与B树

- B树：多路平衡树，每个节点有多个子节点，每个子节点有多个key，每个key对应一个范围，每个范围对应一个子节点
- B+树：B树的一种变种，每个节点只有key，没有数据，每个key对应一个子节点，每个子节点对应一个范围
- B树和B+树的区别：

||BST|B Tree|
|--|--|--|
|比较key|少|多（内存）|
|读取节点|多|少（磁盘）|
|位置|内存|内存+磁盘|


## 存储器的读写速度

- 寄存器    0.1
- 内存      1
- 硬盘      100 ~ 1000
- 网络      100000

BST：    key 4, node 4
BTree:   key 6, node 2

节点的比较开销大
4*1 + 4*100 = 402
6*1 + 2*100 = 206

||`B Tree`|`B+ Tree`|
|--|--|--|
|数据存储|所有节点|叶子节点|
|数据冗余|无|有，内部节点，索引|
|数据访问路径长度|不一致|一致|
|节点之间|没有联系|双向链表，顺序访问|
|区间访问|无|支持|
|场景| - |文件（NTFS（windows的）、ext4（Linux的））、数据库引擎|

# 哈希表（散列表） hash table

- 哈希函数：任意输入，定长的输出；输入确定，输出确定
  - 除留余数法
  - 直接定址 例如学号h=学号-20233547
  - 数字分析法
    平均查找次数ASL（成功）=所有次数/数据数
    平均查找次数ASL（不成功）需要查到一个空位置才确定不存在=所有不成功次数/数据数
  - MD5：已破解
  - SHA128：已破解
  - SHA256：安全，没有哈希冲突，2^256
- 哈希值（存储地址）
- 负载因子
- 哈希冲突
  - 开放定址
    - 线性探测 `*`
    - 平方探测 1 2 4
  - 拉链法
    - 单链表
    - 单链表过长，转为二叉搜索树
