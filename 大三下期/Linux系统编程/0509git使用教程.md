## Git命令

仓库：一个文件夹，被初始化后，由git托管
- 干净状态：没有未提交的文件，也没有未追踪的文件。
- 未追踪状态：文件没有放在缓冲区，文件不在git仓库中，也没有被git跟踪。
- 待提交状态：文件已被修改，但未被提交到git仓库。
- 待同步状态：文件已被提交到git仓库，但未被同步到远程仓库。

### Git命令
- ‘git init’：初始化一个新的Git仓库，会创建隐藏目录.git。
- ‘git status’：显示当前文件的状态。干净、未追踪、待提交、待同步
- ‘git add’：将未追踪的文件添加到暂存区。
- ‘git commit’：提交暂存区的文件到本地仓库。生成了存档及日志。
- ‘git log’：显示提交日志。
- ‘git push’：将本地仓库的提交推送到远程仓库。
- ‘git pull’：远程仓库的最新提交同步到本地仓库。
- ‘git fetch’：从远程仓库拉取最新提交，但不合并到本地仓库。
- ‘git branch’：显示本地分支。
- ‘git merge’：合并指定分支到当前分支。
- ‘git checkout’：切换到指定分支。回到过去的某个版本。
- ‘git checkout -b 新分支名’：创建并切换到新分支。
- ‘git tag’：显示所有标签。
- ‘git tag -a 标签名 -m "标签说明"’：创建一个标签。
## Git顺序

- git init 选一个文件夹进入并使用命令创建一个git仓库

- git add XXX.c
- git commit -m "此次提交做了。。"

- ‘git checkout -b 新分支名’：创建并切换到新分支。
- ‘git checkout’：切换到指定分支。回到过去的某个版本。
- ‘git merge’：合并指定分支到当前分支。

- ‘git push’：将本地仓库的提交推送到远程仓库。
