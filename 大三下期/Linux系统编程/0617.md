# 0617 互斥锁 mutex

## 为什么需要锁？

多个线程共享全局变量时，如果同时读写会产生**竞态条件**（race condition），导致数据错乱。互斥锁保证同一时刻只有一个线程访问共享资源。

## 核心 API

| 函数 | 作用 |
|------|------|
| `pthread_mutex_init` | 初始化锁 |
| `pthread_mutex_lock` | 加锁（阻塞等待） |
| `pthread_mutex_trylock` | 尝试加锁（非阻塞，失败立即返回） |
| `pthread_mutex_unlock` | 解锁 |
| `pthread_mutex_destroy` | 销毁锁 |

## 两种初始化方式

```c
// 方式一：静态初始化（编译期）
pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;

// 方式二：动态初始化（运行时，可设置属性）
pthread_mutex_t mutex;
pthread_mutex_init(&mutex, NULL);
```

## 基本流程

```c
pthread_mutex_lock(&mutex);     // 1. 加锁
// ... 访问共享资源 ...          // 2. 临界区
pthread_mutex_unlock(&mutex);   // 3. 解锁
```

> **注意**：加锁后必须解锁，否则其他线程永久阻塞（死锁）。

## 示例：两个线程轮流写 buffer

```c
// 使用互斥锁保护共享 buffer
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>
#include <string.h>

char buffer[101];                       // 全局共享缓冲区
pthread_mutex_t mutex;                  // 互斥锁

void *worker(void *arg)
{
    long id = (long)arg;

    for (int i = 0; i < 3; i++)
    {
        // 加锁 ———— 进入临界区
        pthread_mutex_lock(&mutex);

        // 操作共享资源
        sprintf(buffer, "线程%ld 第%d次写入", id, i + 1);
        printf("%s\n", buffer);
        sleep(1);                       // 模拟耗时操作

        // 解锁 ———— 离开临界区
        pthread_mutex_unlock(&mutex);

        usleep(100);                    // 让出 CPU，让另一个线程有机会抢锁
    }

    return NULL;
}

int main()
{
    // 初始化锁
    pthread_mutex_init(&mutex, NULL);

    pthread_t t1, t2;
    pthread_create(&t1, NULL, worker, (void *)1);
    pthread_create(&t2, NULL, worker, (void *)2);

    // 等待两个线程结束
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    // 销毁锁
    pthread_mutex_destroy(&mutex);

    return 0;
}
```

### 编译运行

```bash
gcc -o mutex_demo mutex_demo.c -lpthread
./mutex_demo
```

### 预期输出（两个线程交替，不会乱）

```
线程1 第1次写入
线程2 第1次写入
线程1 第2次写入
线程2 第2次写入
线程1 第3次写入
线程2 第3次写入
```

> 如果没有锁，两个线程可能同时写 buffer，输出会混乱。

## 死锁

死锁的典型场景：线程 A 持有锁 1 等待锁 2，线程 B 持有锁 2 等待锁 1，互相等待。

**避免死锁的原则**：
- 加锁和解锁成对出现
- 多锁时保持一致的加锁顺序
- 尽量缩小临界区（锁的持有时间越短越好）
